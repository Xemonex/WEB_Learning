Introduction to Bash

Hi, welcome to my small guide glued for easier beginnings. First what is even bash?
Wikipedia offers this definition:

Bash is a Unix shell and command language written by Brian Fox for the GNU Project as a free software replacement for the Bourne shell.

In my words, I would describe it as a tool that wraps around the kernel and allows you to use and set up a Unix-based operating system. And not only that, as a scripting language, bash can be used to write simple programs.
However, Bash is not the only shell used on Unix-based systems. So how do we know what shell we are currently using? Let's find out. Let's open Terminal first. A window will appear in which there will probably be some such text:

alex@alxcomp:~$

This is a prompt to write what command you want to execute. First part, ie alex, is your username. "@" is a separator that separates the username from the hostname, simply your computer name. It is followed by ":" and is where you are on the file system. You will probably see "~", which is the home directory symbol. The prompt ends with "$", which is a symbol for the standard user, but we'll talk about that in the next lessons.

Now that we know what the prompt means, let's find out if we really use bash. Type "echo $SHELL" and press enter. If you are really in bash, you can see this:

alex@alxcomp:~$ echo $SHELL
/bin/bash

Perfect! You managed your first bash command.

What is around?

Ok, now we know that we are in our home directory because "~" says so. But how do we find out what files and folders are in our home directory. The ls command is used for this. It is based on the English word "listing". And that's exactly what the command will do.

alex@alxcomp:~$ ls
Desktop Documents Downloads Music Pictures Pubic Videos

Exactly according to the job description, the command listed the folders that are in your home folder. And they will probably be very similar to what you know from Windows or Mac OSX. Depending on the Linux flavor you use, folders, files, and possibly different file types may be distinguished by a different text color.

Command "ls" has many other features that we will discuss in other lessons, such as:

  • Can distinguish between files, folders, links and others
  • Allows you to view the permissions on individual files
  • Displays hidden files

Traveling through the filesystem

When we know what folders are around us, let's try to look into some of them. The command "cd" from the English "change directory" is used for this. So let's try to move into the desktop. Type "cd Desktop" and then let's see if there's anything in the desktop.

alex@alxcomp:~$ cd Desktop
alex@alxcomp:~/Desktop$ ls
alex@alxcomp:~/Desktop$

In our case, nothing, so let's try elsewhere. But how do we get back? There are two dots for this. So "cd ..". With this command we get to the folder above. Then let's take a look into the Documents to see if there is anything there. By the way, you may notice that the path has changed at the prompt.

alex@alxcomp:~/Desktop$ cd ..
alex@alxcomp:~/$ cd Documents
alex@alxcomp:~/Documents$ ls
alex@alxcomp:~/Documents$

Nothing again? :/ In the next lesson we are going to fix it.

Let's create things

For creating any file there is command "touch". However, the created file will be empty. Let's start with the file "file.txt". The suffix .txt is not so important for Linux, but it is convenient for easier orientation for users.

alex@alxcomp:~/Documents$ touch file.txt
alex@alxcomp:~/Documents$

Done, it was easy. As you have probably noticed, when the command is executed successfully, a new prompt will appear, without any error message. Now let's just check if we really created a new file. Surely you already know how.

alex@alxcomp:~/Documents$ ls
file.txt
alex@alxcomp:~/Documents$

The file.txt is there! But the file is still empty, how do we fill it? So let's move on to the next chapter.

Less empty world

We encountered the command "echo" in the first chapter. This command forces the bash to tell us what we want. Let's try it. Let's greet the world with the classic "Hello World!". Don't forget the quotes around what we want bash to say.

alex@alxcomp:~/Documents$ echo "Hello World!"
Hello World!
alex@alxcomp:~/Documents$

So the world was greeted. Hmm, but we still have an empty file.txt, how do we fill it? To do this, you need to learn the ">" symbol. If we insert this symbol between what bash "says" and the selected file, then bash will overwrite the contents of this file with exactly what "said" or correctly "output". This type of symbol is called operator, we will talk about others later. Let's do it - "echo" Hello World! "> File.txt".

alex@alxcomp:~/Documents$ echo "Hello World!" > file.txt
alex@alxcomp:~/Documents$

Done. And in the next lesson, we'll look inside the file to see if everything went correctly.

Peeking inside the file

In bash language, we can look inside any file that comes to mind with a simple "cat" command that comes from the English word concatenate. Surely you already know exactly what the command will look like.

alex@alxcomp:~/Documents$ cat file.txt
Hello World!
alex@alxcomp:~/Documents$

Everything works as it should! This way, you can look inside absolutely any file, but it will probably only make sense to you if the file is encoded as plain text. If you look at the picture like this, you will see the machine code. And you probably won't understand it. If so, you are not a human but a robot.
